# 📊 PhonePe Transactions Analysis Dashboard

## 🚀 Project Overview

This project analyzes digital transaction data similar to PhonePe to understand transaction trends, service usage, and failure patterns. The dashboard provides insights into revenue, transaction success rates, and payment issues.

---

## 🛠️ Tools & Technologies

* **Excel** (Data Cleaning & Preparation)
* **Power BI** (Dashboard & Visualization)

---

## 📂 Data Processing (Excel)

* Cleaned raw dataset (handled missing values, formatting issues)
* Standardized columns (date, transaction type, service categories)
* Prepared data for analysis by removing inconsistencies
* Structured data for efficient Power BI modeling

---

## 📊 Dashboard Features

### 🔑 Key KPIs

* Total Transaction Amount
* Total Transactions
* Successful Transactions
* Failed Transactions

---

### 📈 Visualizations

* Service vs Amount (Loans, Insurance, Money Transfer, Recharge & Bills)
* Date vs Transaction Amount (Trend Analysis)
* Failed Payment Reasons Distribution
* Category-wise Transaction Analysis
* Interactive Filters (Date Range, Service Type)

---

## 🎯 Key Insights

* Loans contribute the highest transaction value among all services.
* A significant portion of transactions are successful, with a smaller share of failures.
* Payment failures are mainly due to technical issues like server errors and incorrect PIN.
* Transaction volume fluctuates over time, showing peak periods in certain months.

---

## 💡 Business Value

* Helps identify high-performing services
* Highlights key failure reasons to improve payment success rate
* Supports data-driven decision-making for digital payment platforms

---

## 📌 Conclusion

This project demonstrates data cleaning in Excel and building an interactive Power BI dashboard to analyze transaction behavior and performance.
